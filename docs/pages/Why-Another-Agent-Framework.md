# Why Another Agent Framework?

Who this is for: developers and architects comparing `agent_framework` with other agentic systems.

This page should explain the design philosophy and tradeoffs behind the project.

## Positioning

`agent_framework` focuses on markdown-defined agents, explicit runtime contracts, provider abstraction, orchestration, tracing, and evaluation.

- Why strict decision JSON matters.

- When another framework may be a better fit.
